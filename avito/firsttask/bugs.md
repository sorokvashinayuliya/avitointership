# Найденные баги в тестовом API (qa-internship.avito.com)

## 1. Поле `likes` стало обязательным, но отсутствует в документации
- Эндпоинт: `POST /api/1/item`
- Ожидание по документации: поля `name`, `price`, `sellerId` — обязательные, `likes` — опциональное или отсутствует
- Фактическое поведение: без поля `likes` → `400 Bad Request` с сообщением "поле likes обязательно"
- Любые значения поля (`0`, `"0"`, `[]`, `["1"]`, `{"count": 0}`, `null`) тоже дают 400 с тем же текстом
- Влияние: невозможно создать объявление без знания недокументированного требования
- Встречается у всех кандидатов с ноября 2025 года

## 2. Валидация длины поля `name` не работает
- Сервер принимает строки длиннее 255 символов без ошибки (ожидается 400)
