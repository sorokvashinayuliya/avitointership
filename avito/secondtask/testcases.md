# Функциональные E2E тест-кейсы для таск-трекера

## Общие предусловия
- Пользователь авторизован в системе
- Открыт проект с доской (Kanban/Scrum)
- Базовый URL приложения доступен

---

### TC-01: Создание задачи

**ID:** TC-01  
**Название:** Создание новой задачи через кнопку "Создать"  
**Приоритет:** Критический  
**Предусловия:** Пользователь находится на доске проекта  

| Шаг | Действие | Ожидаемый результат |
|-----|--------|----------------------|
| 1   | Нажать кнопку "Создать задачу" / "+" / "Новая задача" | Открывается модальное окно / форма создания задачи |
| 2   | Ввести в поле "Название" валидное значение (например, "Тестовая задача E2E") | Поле заполняется |
| 3   | (Опционально) Заполнить описание, назначить исполнителя, установить приоритет | Данные отображаются в форме |
| 4   | Нажать кнопку "Создать" / "Сохранить" | Задача создаётся, модальное окно закрывается |
| 5   | Проверить наличие задачи на доске в колонке "To Do" / "Бэклог" | Задача отображается с правильным названием |

**Постусловие:** Задача успешно создана и видна на доске

---

### TC-02: Открытие карточки задачи

**ID:** TC-02  
**Название:** Открытие детальной карточки созданной задачи  
**Приоритет:** Высокий  
**Предусловия:** Задача создана (выполнен TC-01)  

| Шаг | Действие | Ожидаемый результат |
|-----|--------|----------------------|
| 1   | Кликнуть по названию задачи на доске | Открывается модальная карточка задачи |
| 2   | Проверить отображение названия задачи | Название совпадает с введённым |
| 3   | Проверить отображение описания (если было заполнено) | Текст описания корректный |
| 4   | Закрыть карточку (крестик / Esc / клик вне области) | Карточка закрывается, возвращаемся на доску |

---

### TC-03: Поиск задачи

**ID:** TC-03  
**Название:** Поиск задачи по точному названию  
**Приоритет:** Высокий  
**Предусловия:** Задача создана и видна на доске  

| Шаг | Действие | Ожидаемый результат |
|-----|--------|----------------------|
| 1   | Открыть глобальный поиск (Ctrl+K / иконка лупы) | Появляется поле ввода поиска |
| 2   | Ввести точное название созданной задачи | В результатах поиска отображается нужная задача |
| 3   | Кликнуть по найденной задаче | Открывается карточка задачи |
| 4   | Проверить, что открыта правильная задача | Название в карточке совпадает с искомым |

---

### TC-04: Переход на доску проекта

**ID:** TC-04  
**Название:** Переход на доску конкретного проекта из бокового меню  
**Приоритет:** Средний  

| Шаг | Действие | Ожидаемый результат |
|-----|--------|----------------------|
| 1   | Нажать на пункт меню "Проекты" / "Доски" | Открывается список доступных проектов |
| 2   | Выбрать нужный проект из списка | Происходит переход на Kanban-доску выбранного проекта |
| 3   | Проверить URL (должен содержать идентификатор проекта) | URL обновлён корректно |
| 4   | Убедиться, что отображаются задачи проекта | На доске видны задачи, включая ранее созданную |

---

### TC-05: Перемещение задачи по статусам (Drag & Drop)

**ID:** TC-05  
**Название:** Перевод задачи из "To Do" в "Done" через перетаскивание  
**Приоритет:** Критический  
**Предусловия:** Задача находится в колонке "To Do"  

| Шаг | Действие | Ожидаемый результат |
|-----|--------|----------------------|
| 1   | Перетащить задачу из колонки "To Do" в "In Progress" | Задача перемещается в новую колонку |
| 2   | Перетащить задачу из "In Progress" в "Done" | Задача отображается в колонке "Done" |
| 3   | Обновить страницу (F5) | Задача остаётся в колонке "Done" |
| 4   | Открыть карточку задачи | В статусе отображается "Done" |

---

## Итог: покрытые пользовательские сценарии
- Создание задачи
- Открытие карточки задачи
- Поиск задачи
- Переход на доску проекта
- Перемещение задачи по статусам

Все тест-кейсы проверяют **результат действий пользователя** и соответствуют требованиям E2E-тестирования.
